# 그림 놀이터 v2 (테마 + QR코드 버전)

메인 컴퓨터에서 테마(바다/하늘/숲속/우주)를 고르면, 화면 안에서 바로 QR코드가 뜨고, 아이들이 그 QR을 태블릿/폰으로 찍어 들어가서 그림을 그리면 큰 화면에 테마에 어울리는 모습으로 떠다니는 프로그램입니다. 완전 무료이고, 같은 와이파이 안에서만 동작합니다.

v1과 달라진 점: 터미널 없이 화면에서 QR코드 확인, 테마별 배경/움직임, 놀이터 제목 입력, 화면의 "나가기" 버튼으로 그림 전체 삭제 후 테마 선택 화면으로 복귀.

## 준비물

- 큰 화면에 연결된 컴퓨터 1대 (서버 겸 디스플레이 역할)
- 아이들 태블릿/폰 여러 대, 컴퓨터와 **같은 와이파이**
- [Node.js](https://nodejs.org) 설치 (최초 1회)

## 설치 (최초 1회)

```
cd kids-drawing-wall-v2
npm install
```

## 실행

```
npm start
```

터미널에 이런 안내가 뜹니다.

```
메인 컴퓨터(큰 화면)에서 열기: http://192.168.x.x:3000/
```

1. **메인 컴퓨터** 브라우저에서 위 주소를 열고 전체화면(F11)으로 전환합니다.
2. 제목(선택)을 입력하고 테마를 고른 뒤 "시작하기"를 누릅니다.
3. 큰 화면 아래쪽 "➕ 그림 추가하기" 버튼을 누르면 QR코드가 뜹니다.
4. 아이들이 **태블릿/폰 카메라**로 QR을 찍어 그림 그리는 화면으로 들어갑니다. (같은 와이파이 필요)
5. 그림을 그리고 "화면으로 보내기"를 누르면 몇 초 안에 큰 화면에 나타나 테마에 맞게 움직입니다.
   - 바다: 물고기처럼 좌우로 헤엄치듯 이동
   - 하늘: 풍선처럼 위로 둥실둥실
   - 숲속 / 우주: 화면 전체를 자유롭게 떠다님
6. 다 놀았으면 메인 화면 우측 상단 "나가기"를 누르면 그림이 모두 사라지고 테마 선택 화면으로 돌아갑니다.

## 자주 겪는 문제

- **태블릿에서 QR을 찍었는데 안 열려요** → 컴퓨터와 태블릿이 같은 와이파이인지 확인하세요. 공용/회사 와이파이는 기기 간 통신이 막힌 경우가 많아 스마트폰 핫스팟을 쓰는 게 가장 확실합니다.
- **QR 대신 주소를 직접 입력하고 싶어요** → QR 모달 안에 주소 텍스트도 함께 표시되니 그걸 입력해도 됩니다.
- **그림이 너무 많이 쌓여요** → 최대 35개까지만 유지되고 오래된 것부터 자동으로 사라집니다. `public/display.html`의 `MAX_ITEMS`로 조절 가능합니다.
- **포트 3000이 이미 사용 중이에요** → `PORT=4000 npm start` 처럼 다른 포트로 실행하세요.

## 나중에 인터넷 웹사이트로 옮기고 싶다면

지금 구조를 거의 그대로 Render, Railway 같은 호스팅 서비스에 올리면 됩니다. 딱 한 가지, 서버가 QR코드/주소를 만들 때 사용하는 기준 주소를 환경변수로 지정해주면 됩니다.

```
BASE_URL=https://내도메인.com npm start
```

이렇게 하면 로컬 IP 대신 지정한 주소로 QR코드와 링크가 만들어집니다. 코드 수정 없이 배포 환경만 바꾸면 되는 구조입니다.

### Render.com에 배포하는 체크리스트

이 순서대로 그대로 따라 하면 배포가 끝납니다.

1. **GitHub 저장소 만들기**
   - [github.com/new](https://github.com/new)에서 새 저장소 생성 (이름 예: `kids-drawing-wall`, Public/Private 아무거나)
   - "Add a README" 등 옵션은 체크하지 않고 빈 저장소로 생성
   - 이 프로젝트 폴더에서 아래 명령 실행 (README에 실제 저장소 주소로 바꿔서):
     ```
     git remote add origin https://github.com/내계정/kids-drawing-wall.git
     git branch -M main
     git push -u origin main
     ```

2. **Render 계정 가입 및 저장소 연결**
   - [render.com](https://render.com)에서 GitHub 계정으로 가입/로그인
   - 대시보드에서 "New +" → "Web Service" 선택
   - 방금 만든 GitHub 저장소를 선택 (Render가 저장소 접근 권한을 요청하면 승인)
   - 이 저장소에 `render.yaml`이 있으므로 Render가 설정을 자동으로 인식합니다 (Runtime: Node, Build: `npm install`, Start: `npm start`)

3. **환경변수 등록**
   - Render 서비스 화면의 "Environment" 탭에서 `BASE_URL` 값을 입력
   - 처음엔 Render가 배정한 기본 주소(`https://서비스이름.onrender.com`)로 입력하고, "Create Web Service" 또는 "Deploy" 실행
   - (선택) `SESSION_TTL_HOURS`를 다르게 쓰고 싶으면 값 변경 (기본 12)

4. **배포 확인**
   - 배포가 끝나면 Render가 준 주소(`https://서비스이름.onrender.com`)로 접속해 테마 선택 화면이 뜨는지 확인
   - "그림 추가하기"로 QR코드를 띄우고, 실제 휴대폰으로 스캔해서 `https://...`로 열리는지, 그림을 보내면 화면에 뜨는지 확인

5. **무료 티어 참고사항**
   - Render 무료 웹서비스는 일정 시간 요청이 없으면 슬립 모드로 들어가고, 다음 요청 시 첫 응답이 몇십 초 걸릴 수 있습니다 (수업/행사 시작 전에 한 번 미리 접속해서 깨워두는 것을 추천)
   - 서버가 재시작되면 메모리에 있던 세션(진행 중인 놀이터)은 초기화됩니다 (원래 설계상 그림도 저장하지 않으므로 동일한 동작)

6. **로컬 사용은 그대로 가능**
   - 배포 후에도 `BASE_URL`을 지정하지 않고 `npm start`로 실행하면 지금처럼 같은 와이파이 안에서 로컬 IP로 동작합니다

## 향후 확장 아이디어 (아직 미구현)

- 테마 선택 화면에서 원하는 주제를 직접 입력하면 AI가 배경 이미지를 생성해주는 기능 (OpenAI 등 이미지 생성 API 연동 필요, 비용 발생)
- 그림을 서버에 잠깐 저장해서 놀이 종료 후 사진첩처럼 모아보기
- 여러 메인 화면(교실 여러 곳)이 동시에 독립적으로 운영되는 것은 이미 세션 구조상 가능 — 각자 다른 테마 선택 화면을 열면 별도로 동작합니다.

## 커스터마이징 팁

- `public/themes.js`: 테마 색상/그라디언트 정의
- `public/display.html`: 테마별 배경 장식(`createDecorations`), 움직임 물리(`tick`), 최대 그림 개수(`MAX_ITEMS`)
- `public/draw.html`: 색상 팔레트, 브러시 굵기
